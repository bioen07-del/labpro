## Usage

Git workflow, деплой в Vercel, работа с Supabase.

## Steps

1. Всегда используй ТОЛЬКО live данные из Supabase, НИКАКИХ mock-данных
2. Git команды в PowerShell: `git add -A; git commit -m "feat/fix: описание"; git push origin master`
3. Проверяй Vercel деплой через MCP команды
4. Читай логи ошибок и исправляй СТРОГО по ним
