LabPro cycle# LabPro Rules & Workflow

## Рабочий Цикл

| Этап | Исполнитель | Описание |
|------|-------------|----------|
| 1 | **Пользователь** | Получение задания |
| 2 | **Агент (Cline)** | Подготовка плана → Подтверждение |
| 3 | **Пользователь** | Одобрение плана (Act mode) |
| 4 | **Агент** | Работа: код, изменения |
| 5 | **Агент** | Деплой: git commit → git push → Vercel |
| 6 | **Агент** | Проверка статуса деплоя и логов |
| 7 | **Пользователь** | Проверка сборки в браузере |
| 8 | **Пользователь** | Обратная связь / новые задания |

## Обязательные Правила для Агента

### 1. Подтверждение Задания
Для **каждого** задания агент **обязан**:
- Повторить задание своими словами
- Перечислить планируемые действия
- Указать ожидаемые результаты

### 2. Работа с Логами
- **ВСЕГДА** смотреть вывод команд и логи
- **ВСЕГДА** реагировать на ошибки (перезапуск, исправление)
- **НЕМЕДЛЕННО** сообщать о проблемах

### 3. Деплой и Сборка
- Проверять статус сборки
- Читать ошибки из логов Vercel
- **Исправлять СТРОГО по логам**, а не гадать
- Не менять код без проверки логов

### 4. Git Workflow
1. `git add -A`
2. `git commit -m "feat/fix: описание"`
3. `git push origin master`
4. Vercel автоматически деплоит

### 5. Vercel MCP
- Использовать `vercel_createdeployment` для ручного деploy
- Использовать `vercel_getdeployments` для проверки статуса
- Ждать статуса `READY` перед следующими действиями

## Пример Подтверждения Задания

```
Я понял задание: исправить TypeScript ошибку в файле tasks/page.tsx
Поле 'culture_id' не существует в типе Task.

План:
1. Проанализировать тип Task в types/index.ts
2. Добавить culture_id в mock данные или использовать task.culture.id
3. Протестировать локально: npm run build
4. Git commit и push
5. Проверить Vercel деплой

Ожидаемый результат: сборка проходит без ошибок
```

## Правила для VS Code AI Assistant

1. Читать этот файл перед началом работы
2. Следовать рабочему циклу
3. Всегда проверять логи команд
4. Сообщать о проблемах сразу
